# Multica APP 部署文档

## 环境要求

- JDK 17+
- Android SDK（通过 `gradlew.bat` 自动查找，或设置 `ANDROID_HOME` 环境变量）
- Python 3.8+（用于自动安装到设备）
- pip 包：`pure-python-adb`（`pip install pure-python-adb`）
- USB 数据线 + 手机开启 USB 调试

## 编译

### 方式 1：命令行编译

```powershell
cd "项目根目录"

# 编译 debug APK（所有 ABI）
.\gradlew.bat assembleDebug
```

编译产物在 `app\build\outputs\apk\debug\` 下：

| 文件 | 适用架构 |
|------|---------|
| `app-arm64-v8a-debug.apk` | 主流 ARM 手机（三星、小米等） |
| `app-armeabi-v7a-debug.apk` | 旧款 ARM 手机 |
| `app-x86-debug.apk` | x86 模拟器 / 平板 |
| `app-x86_64-debug.apk` | x86_64 模拟器 |
| `app-universal-debug.apk` | 通用（体积最大，兼容所有架构） |

### 方式 2：Android Studio

1. 打开项目
2. **Build → Make Project** (Ctrl+F9)
3. 产物同上

### 常见编译问题

**OneDrive 文件锁导致 `AccessDeniedException`**：
```powershell
# 先删除 build 缓存再编译
Remove-Item -Recurse -Force app\build -ErrorAction SilentlyContinue
.\gradlew.bat assembleDebug
```

## 安装到设备

### 方式 1：Python 自动安装（推荐，自动匹配架构）

```powershell
# 首次使用需安装依赖
pip install pure-python-adb

# 编译 + 安装一步到位
.\gradlew.bat assembleDebug
python install_usb.py
```

脚本会自动：
- 连接 ADB server（127.0.0.1:5037）
- 检测每个设备的 CPU 架构
- 安装对应的 APK（arm64 设备装 arm64 APK，x86 设备装 x86 APK）
- 如果覆盖安装失败，自动卸载后重新安装

### 方式 2：ADB 手动安装

```powershell
# 查看已连接设备
adb devices -l

# 单设备直接安装
adb install -r app\build\outputs\apk\debug\app-arm64-v8a-debug.apk

# 多设备需指定序列号
adb -s ce07171712dcc228027e install -r app\build\outputs\apk\debug\app-arm64-v8a-debug.apk
adb -s 53P021NA01 install -r app\build\outputs\apk\debug\app-x86-debug.apk
```

### 方式 3：手动传 APK

1. 找到 `app\build\outputs\apk\debug\` 下的 APK 文件
2. 通过微信、邮件、USB 拷贝到手机
3. 手机上打开文件管理器安装

## 一键编译+安装脚本

项目根目录下有以下脚本：

| 脚本 | 用途 |
|------|------|
| `build-and-install.bat` | Windows bat 脚本，编译+安装（有编码问题，不推荐） |
| `install-to-devices.ps1` | PowerShell 安装脚本 |
| `install-to-devices.bat` | bat 安装脚本 |
| `install_samsung.ps1` | 仅安装三星设备 |
| `install_usb.py` | Python 自动安装（推荐） |

### 推荐工作流

```powershell
# 1. 编译
.\gradlew.bat assembleDebug

# 2. 安装到所有设备
python install_usb.py
```
